# **\---**

# 

# **\#\# 1\) Principios de outputs**

# **\- Todo output es versionado (nunca se pisa)**

# **\- Dos tipos de versiones:**

#   **\- IA (generated)**

#   **\- Manual edit (edited)**

# **\- Regeneración SIEMPRE usa:**

#   **\- inputs base de etapa**

#   **\- feedback de regeneración (nuevo input)**

#   **\- “elementos a mantener” (checklist)**

# **\- Aprobación “congela” una versión como referencia para etapas siguientes**

# 

# **\---**

# 

# **\#\# 2\) Alternativas por etapa**

# **Cada etapa puede generar:**

# **\- 1 output (modo simple)**

# **\- o “Batch de alternativas” (3–5) en un solo run**

# **El usuario puede:**

# **\- seleccionar 1 para continuar**

# **\- o clonar proyecto para explorar otra “realidad” (branch)**

#

# **\#\# 3\) Módulo A — Creación de Marca (entregables)**

# **Outputs por etapas (resumen):**

# **\- A1: posicionamiento \+ propuesta de valor \+ público \+ tono**

# **\- A2: propuestas naming \+ shortlist \+ validación interna**

# **\- A3: manifiesto \+ narrativa \+ elevator pitch**

# **\- A4: propuestas de logo \+ variantes \+ sistema visual**

# **\- A5: aplicaciones (papelería, social, web/landing) \+ mockups**

# **\- A6: Brand Pack final**

# **\#\# 4\) Módulo B — Brand Strategy (entregables)**

# **Strategy Pack (PDF \+ MD):**

# **\- Diagnóstico estratégico**

# **\- Consumer tensions \+ JTBD \+ barreras/motivadores**

# **\- Mapa competitivo \+ frames \+ whitespace**

# **\- Cascada de elecciones (CSO): dónde jugar / cómo ganar / trade-offs**

# **\- Sistema de métricas: KPIs \+ targets \+ triggers de acción**

# **\- Narrativa: territorio \+ pitches \+ historia**

# **\- Plan de batalla (prioridades 30/60/90 días)**

# **\- Ronda de verificación (resumen de checks)**
